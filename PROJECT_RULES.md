# PROJECT_RULES

## 1. 当前开发阶段
这是一个 MVP 项目，当前进入 **Web 登录版准备阶段**。
目标是在不破坏现有稳定功能的前提下，逐步支持多用户架构。

## 2. 当前阶段允许实现的能力
- 用户登录系统
- 后端 API
- 数据库存储
- 用户级数据隔离

## 3. 当前阶段仍不做的能力
- 不接 AI API
- 不做复杂动画
- 不做过度工程化改造

## 4. 开发原则
- 小步迭代
- 尽量最小改动
- 不随意重构整个项目
- 优先保持代码简单
- 保持最小可运行闭环

## 5. Web 阶段约束
- 不改变核心复习算法逻辑（Review Algorithm v1）
- 不重写前端页面结构
- 不引入复杂框架，保持轻量
- 新增后端能力必须可渐进接入，不影响当前 MVP 稳定性

## 6. 数据结构概念（抽象）
- `User`
- `Words`
- `ReviewSession`
- `ReviewHistory`

> 说明：所有学习数据都应以用户为边界进行归属和隔离。

## 7. Web 架构概念（抽象）
- `Frontend (React)`
- `Backend API`
- `Database`

> 说明：文档层定义架构方向，具体技术实现按迭代阶段逐步确定。

## 8. AI 协作规则
- 每次修改代码前先说明计划
- 修改完成后必须说明修改了哪些文件
- 不引入新的大型依赖库（如确有必要需先说明理由）

## 9. 每次任务完成后需要说明
- 修改了哪些文件
- 每个文件做了什么改动
- 如何运行验证
- 下一步开发建议

## 10. 单词分级规则（正式版）
分级不是单纯按词本身难度判断，而是按“这个词对当前学习目标的重要性 + 希望掌握到什么程度”来判断。

### A级词汇
- 需要深度掌握并能够主动、稳定输出
- 能准确理解核心含义及常见延伸义
- 能主动用于口语和写作
- 能掌握常见搭配、语气色彩和适用语境
- 能区分与近义词之间的重要差异
- 通常是值得重点反复训练的词

### B级词汇
- 需要较熟练理解，并具备一定主动使用能力
- 能理解主要含义
- 在看到或听到时能较稳定识别
- 在写作或口语中有机会主动使用，但不要求像A级那样深度掌握
- 对常见搭配和语境有基本认识
- 可以作为中期提升对象，逐步向A级过渡

### C级词汇
- 以识别和基础理解为主，暂不要求高频主动输出
- 看到时大致知道意思
- 能完成基本理解，不严重影响阅读或听力
- 不要求当前阶段稳定主动使用
- 不要求深入掌握搭配、语体和细微差别
- 后续如果重要性提升，可以再升到B或A
